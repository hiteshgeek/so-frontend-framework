DataTable Plugin

Complete Developer Documentation - A powerful, feature-rich data table component

infoOverview

The DataTable plugin provides a powerful, feature-rich table component built on Tabulator with Material UI styling. It includes advanced features like row expansion, cell selection, column pinning, grouping, export functionality, and seamless API integration.

add_circleRow Expand

Expand rows to show detailed content loaded dynamically

check_boxRow Selection

Multi-row selection with checkboxes and callbacks

push_pinColumn Pinning

Pin columns to left or right for easy reference

group_workMulti-Level Grouping

Group data by multiple columns with drag reorder

downloadExport

Export to Excel/CSV with custom options

paletteColumn Colors

Apply aesthetic colors to highlight columns

add_circleRow Expand (Plus Icon)

Enable expandable rows to show additional details. When enabled, a plus (+) icon appears as the first column. Clicking it expands the row to display custom HTML content.

Configuration

codeJavaScript 
const dataTable = new DataTable('container', {
    columns: sampleColumns,
    data: sampleData,
    enableRowExpand: true,  // Enable the plus icon column

    // Callback when row is expanded
    onRowExpand: function(rowData, callback) {
        // rowData: The data object for the expanded row
        // callback: Function to call with HTML content to display

        // Option 1: Simple synchronous content
        const html = `
            <div class="dt-expand-content">
                <h3>Details for ${rowData.name}</h3>
                <p>Email: ${rowData.email}</p>
                <p>Department: ${rowData.department}</p>
            </div>
        `;
        callback(html);
    }
});

Loading Dynamic Content via AJAX

You can fetch additional data from an API before displaying the expanded content:

codeJavaScript 
onRowExpand: function(rowData, callback) {
    // Show loading state
    callback('<div class="dt-expand-loading">Loading...</div>');

    // Fetch additional data from API
    fetch(`/api/employees/${rowData.id}/details`)
        .then(response => response.json())
        .then(details => {
            const html = `
                <div class="dt-expand-content">
                    <h3>Employee Details</h3>
                    <div class="dt-detail-grid">
                        <div class="dt-detail-item">
                            <span class="dt-detail-label">Full Name</span>
                            <span class="dt-detail-value">${details.fullName}</span>
                        </div>
                        <div class="dt-detail-item">
                            <span class="dt-detail-label">Phone</span>
                            <span class="dt-detail-value">${details.phone}</span>
                        </div>
                        <div class="dt-detail-item">
                            <span class="dt-detail-label">Address</span>
                            <span class="dt-detail-value">${details.address}</span>
                        </div>
                        <div class="dt-detail-item">
                            <span class="dt-detail-label">Manager</span>
                            <span class="dt-detail-value">${details.manager}</span>
                        </div>
                    </div>
                </div>
            `;
            callback(html);
        })
        .catch(error => {
            callback('<div class="dt-expand-error">Failed to load details</div>');
        });
}

Tip: The expand icon column is automatically frozen to the left side of the table for easy access when scrolling horizontally.

check_boxCheckbox & Row Selection

Enable row selection with checkboxes. Users can select individual rows or use the header checkbox to select all rows on the current page.

Configuration

codeJavaScript 
const dataTable = new DataTable('container', {
    columns: sampleColumns,
    data: sampleData,
    enableRowSelection: true,  // Enable checkbox column

    // Callback when selection changes
    onRowSelectionChange: function(selectedData, selectedRows) {
        // selectedData: Array of data objects for selected rows
        // selectedRows: Array of Tabulator row components

        console.log('Selected count:', selectedData.length);
        console.log('Selected IDs:', selectedData.map(row => row.id));

        // Example: Enable/disable bulk action buttons
        document.getElementById('deleteBtn').disabled = selectedData.length === 0;
        document.getElementById('exportBtn').disabled = selectedData.length === 0;
    }
});

Programmatic Selection

codeJavaScript 
// Get currently selected rows
const selected = dataTable.getSelectedRows();

// Select all rows
dataTable.selectAllRows();

// Deselect all rows
dataTable.deselectAllRows();

// Get selected data
const selectedData = dataTable.getSelectedData();

Checkbox States

The header checkbox supports three states:

  • Unchecked: No rows selected
  • Checked: All visible rows selected
  • Indeterminate: Some rows selected (partial selection)

more_vertAction Buttons

Add action buttons to each row for operations like View, Edit, Delete. Actions appear in a dropdown menu triggered by a three-dot icon.

Defining Actions

codeJavaScript 
const dataTable = new DataTable('container', {
    columns: sampleColumns,
    data: sampleData,

    // Define action buttons
    actions: [
        {
            id: 'view',
            label: 'View',
            icon: 'visibility',  // Material icon name
            onClick: function(rowData) {
                console.log('View clicked for:', rowData);
                window.location.href = `/employees/${rowData.id}`;
            }
        },
        {
            id: 'edit',
            label: 'Edit',
            icon: 'edit',
            onClick: function(rowData) {
                console.log('Edit clicked for:', rowData);
                openEditModal(rowData);
            }
        },
        {
            id: 'delete',
            label: 'Delete',
            icon: 'delete',
            onClick: function(rowData) {
                if (confirm(`Delete ${rowData.name}?`)) {
                    deleteEmployee(rowData.id);
                }
            }
        }
    ],

    // Optional: Global action callback
    onActionClick: function(action, rowData) {
        console.log('Action clicked:', action.label, 'for row:', rowData);
        // Use for logging, analytics, etc.
    }
});

cloud_downloadLoading Data from API

Load data from an API endpoint instead of providing static data. The plugin will fetch and display the data automatically.

Basic API Loading

codeJavaScript 
const dataTable = new DataTable('container', {
    columns: sampleColumns,
    dataUrl: 'https://api.example.com/employees',  // API endpoint
    filters: {
        department: 'Sales',
        status: 'Active',
        minSalary: 50000
    }
});

Expected API Response Format

codeJSON 
{
    "data": [
        { "id": 1, "name": "John Doe", "salary": 75000 },
        { "id": 2, "name": "Jane Smith", "salary": 82000 }
    ],
    "totals": {
        "salary": 157000,
        "bonus": 25000

    "meta": {
        "total": 100,

Note: The API should return data in the expected format. If your API returns a different structure, you may need to transform the response.

refreshReload Data on Filter Change

When using API-based data, you can reload the table with new filters when the user changes filter options.

Using setFilters Method

codeJavaScript 
// Initialize table with API
const dataTable = new DataTable('container', {
    columns: sampleColumns,
    dataUrl: '/api/employees',
    filters: { department: 'All' }
});

// When user changes department filter
document.getElementById('departmentFilter').addEventListener('change', function(e) {
    dataTable.setFilters({
        department: e.target.value,
        status: document.getElementById('statusFilter').value
    });
    // Table automatically reloads with new filters
});

Manual Reload

codeJavaScript 
// Reload with current filters
dataTable.refresh();

// Or set new data directly (for local data)
dataTable.setData(newDataArray);

Complete Filter Example

codeJavaScript 
// HTML: Filter controls
// <select id="dept-filter">...</select>
// <select id="status-filter">...</select>
// <button id="apply-filters">Apply</button>

document.getElementById('apply-filters').addEventListener('click', function() {
    const filters = {
        department: document.getElementById('dept-filter').value,
        status: document.getElementById('status-filter').value,
        dateFrom: document.getElementById('date-from').value,
        dateTo: document.getElementById('date-to').value
    };

    // Remove empty filters
    Object.keys(filters).forEach(key => {
        if (!filters[key] || filters[key] === 'all') {
            delete filters[key];
        }
    });

    // Reload table with new filters
    dataTable.setFilters(filters);
});

searchSearch with API Data

The built-in search works client-side by default. For API-based data, you can integrate server-side search.

Client-Side Search (Default)

When enableSearch: true, the search box filters data already loaded in the table. This works well for smaller datasets.

Server-Side Search

For large datasets, implement server-side search by listening to search input and reloading data:

codeJavaScript 
const dataTable = new DataTable('container', {
    columns: sampleColumns,
    dataUrl: '/api/employees',
    enableSearch: true,

    // Custom search handler for API
    onSearch: function(query) {
        // Debounce to avoid too many API calls
        clearTimeout(this.searchTimeout);
        this.searchTimeout = setTimeout(() => {
            // Add search query to filters and reload
            this.setFilters({
                ...this.options.filters,
                search: query
            });
        }, 300);
    }
});

// Alternative: Manual search implementation
const searchInput = document.querySelector('[data-search]');
let searchTimeout;

searchInput.addEventListener('input', function(e) {
    clearTimeout(searchTimeout);
    searchTimeout = setTimeout(() => {
        const query = e.target.value;

        if (query.length >= 2 || query.length === 0) {
            dataTable.setFilters({
                ...dataTable.options.filters,
                search: query
            });
        }
    }, 300);
});

Tip: Always debounce search input to avoid excessive API calls while the user is typing.

all_inclusiveInfinite Scroll

Disable pagination to enable infinite scroll mode. All data loads at once and users can scroll through the entire dataset.

Enable Infinite Scroll

codeJavaScript 
const dataTable = new DataTable('container', {
    columns: sampleColumns,
    data: sampleData,        // Full dataset
    pagination: false,       // Disable pagination = infinite scroll

    // Other options work normally
    enableSearch: true,
    enableColumnSelection: true,
    showFooter: true
});

Infinite Scroll with API

codeJavaScript 
const dataTable = new DataTable('container', {
    columns: sampleColumns,
    dataUrl: '/api/employees?limit=1000',  // Load all data
    pagination: false
});

Performance Note: Infinite scroll works best with datasets under 10,000 rows. For larger datasets, consider using pagination or virtual scrolling.

cloud_uploadAPI-Based Export

For server-side export, post export configuration to an API endpoint and receive the generated file.

Custom Export URL

codeJavaScript 
const dataTable = new DataTable('container', {
    columns: sampleColumns,
    dataUrl: '/api/employees',

    // Custom export endpoint
    exportUrl: '/api/employees/export',

    exportOptions: [
        { key: 'includeHeader', label: 'Include headers', type: 'checkbox', default: true },
        { key: 'format', label: 'File format', type: 'text', default: 'xlsx' }
    ],

    // Callback to customize export request
    onExport: function(format, options, columns) {
        // format: 'excel' or 'csv'
        // options: User-selected export options
        // columns: Array of selected column definitions

        const exportData = {
            format: format,
            options: options,
            columns: columns.map(col => col.field),
            filters: this.options.filters,  // Current filters
            sort: this.getCurrentSort()     // Current sort
        };

        // Post to export API
        fetch('/api/employees/export', {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify(exportData)
        })
        .then(response => response.blob())
        .then(blob => {
            // Download the file
            const url = window.URL.createObjectURL(blob);
            const a = document.createElement('a');
            a.href = url;
            a.download = `export_${Date.now()}.${format === 'excel' ? 'xlsx' : 'csv'}`;
            a.click();
            window.URL.revokeObjectURL(url);
        });

        return false;  // Prevent default client-side export
    }
});

Export Request Format

codeJSON 
{
    "format": "excel",
    "options": {
        "includeHeader": true,
        "alternateRowColors": true,
        "sheetName": "Employees"
    },
    "columns": ["id", "name", "email", "department", "salary"],
    "filters": {
        "department": "Sales",
        "status": "Active"
    },
    "sort": {
        "field": "name",
        "direction": "asc"
    }
}

saveGet Current Configuration

Retrieve the current table configuration including column order, visibility, pinning, and other settings that the user has modified.

Get Configuration

codeJavaScript 
// Get current table configuration
const config = dataTable.getConfig();

console.log(config);
// {
//     columns: [...],           // Current column definitions with order
//     pinnedColumns: {...},     // { fieldName: 'left'|'right' }
//     columnVisibility: {...},  // { fieldName: true|false }
//     columnOrder: [...],       // Array of field names in current order
//     columnWidths: {...},      // { fieldName: width }
//     wrapSettings: {...},      // { fieldName: 'wrap'|'clip' }
//     groupBy: [...],           // Current grouping columns
//     cellSpacing: 'normal',    // Current spacing setting
//     verticalAlign: 'top'      // Current vertical alignment
// }

Save and Restore Configuration

codeJavaScript 
// Save configuration to localStorage
function saveTableConfig() {
    const config = dataTable.getConfig();
    localStorage.setItem('myTableConfig', JSON.stringify(config));
}

// Save button handler
document.getElementById('saveConfigBtn').addEventListener('click', saveTableConfig);

// Restore configuration on page load
function loadTableConfig() {
    const savedConfig = localStorage.getItem('myTableConfig');
    if (savedConfig) {
        const config = JSON.parse(savedConfig);

        // Apply saved configuration when initializing
        const dataTable = new DataTable('container', {
            columns: config.columns,
            data: sampleData,
            groupBy: config.groupBy[0] || null,
            cellSpacing: config.cellSpacing,
            cellVerticalAlign: config.verticalAlign
            // ... other options
        });

        // Apply column settings after initialization
        dataTable.applyConfig(config);
    }
}

Send Configuration to Server

codeJavaScript 
// Save configuration to server
async function saveConfigToServer() {
    const config = dataTable.getConfig();

    await fetch('/api/user/table-config', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
            tableId: 'employees-table',
            config: config
        })
    });
}

// Listen for config changes
dataTable.onConfigChange = function() {
    // Auto-save when user makes changes
    saveConfigToServer();
};

boltEvents & Callbacks

Handle various table events with callback functions.

CallbackParametersDescription
onRowClick rowData Called when a row is clicked
onRowSelectionChange selectedData, selectedRows Called when row selection changes
onRowExpand rowData, callback Called when expand icon is clicked
onActionClick action, rowData Called when any action button is clicked
onExport format, options, columns Called before export (return false to prevent)
onSearch query Called when search input changes
onConfigChange none Called when table configuration changes

Example

codeJavaScript 
const dataTable = new DataTable('container', {
    columns: sampleColumns,
    data: sampleData,

    onRowClick: function(rowData) {
        console.log('Row clicked:', rowData.id);
    },

    onRowSelectionChange: function(selectedData, selectedRows) {
        document.getElementById('selected-count').textContent = selectedData.length;
    },

    onRowExpand: function(rowData, callback) {
        fetch(`/api/details/${rowData.id}`)
            .then(r => r.json())
            .then(data => callback(renderDetails(data)));
    },

    onActionClick: function(action, rowData) {
        console.log(`${action.label} clicked for ${rowData.name}`);
    }
});

Recommended Usage
